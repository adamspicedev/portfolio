## Spacing Guidelines

- Use single line spacing between:

  - Imports groups (e.g., `scala.concurrent.Future`)
  - Functions

- Use double line spacing between the end of imports and the start of a class.

- Blank line before the line or block that returns from a function

  - Youâ€™re doing this for ~80% of your functions
  - Some remaining to be done (def call, def renderRowOption etc)

  - def onDidUpdate is nice as it is, because the `if` block is essentially the contents of the if block does side effects rather than returning something.

  - No spacing required when chaining a single statement i.e.
    ```scala
    thing.filter(_.otherThing)
    	.map(_.something)
    	.map(<.div(_))
    	.toVdomArray
    ```

- No brackets around a single statement function
- No spaces between closing brackets i.e.

```scala
// BAD
statement ) )

)

)

// GOOD
statement))))
```
